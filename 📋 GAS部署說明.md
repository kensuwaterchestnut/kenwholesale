# 📋 GAS 完整部署說明

## ⚠️ 重要發現

**之前的 GAS 程式碼缺少了寄信函數！** 這就是為什麼下單後沒有收到郵件的原因。

現在已經補齊，共有 **4 個檔案**需要依序複製到 GAS 編輯器。

---

## 🚀 部署步驟

### 1️⃣ 開啟 Google Apps Script 編輯器
1. 打開您的 Google 試算表
2. 點選「擴充功能」→「Apps Script」

### 2️⃣ 依序複製貼上程式碼

**按照以下順序，將 4 個檔案的內容全部複製貼上到 GAS 編輯器中：**

```
Part1_常數和基礎函數.gs    ← 第一部分（常數、doGet、doPost）
Part2_訂單查詢和編輯.gs    ← 第二部分（訂單查詢、onEdit）
Part3_最後工具函數.gs      ← 第三部分（統計報表）
Part4_寄信函數.gs          ← 第四部分（✅ 新增！寄信功能）
```

**⚠️ 注意：** 直接全部貼在同一個 `Code.gs` 檔案中即可，不需要分開建立多個檔案。

### 3️⃣ 儲存並部署

1. 點選「儲存」圖示 💾
2. 點選「部署」→「新增部署作業」
3. 類型選擇「網頁應用程式」
4. 設定：
   - **執行身分**：我
   - **具有存取權的使用者**：所有人
5. 點選「部署」
6. 複製「網頁應用程式網址」（這就是您的 GAS API 網址）

### 4️⃣ 授權並安裝觸發器

1. 重新整理試算表頁面
2. 頂部選單會出現「啃酥面板」
3. 點選「啃酥面板」→「授權並安裝」
4. 依照提示完成授權（需允許「傳送電子郵件」權限）

## 🔧 常見問題

### Q1: 為什麼之前沒有收到郵件？
**A:** Part1-3 的程式碼中缺少了寄信函數的實作（`sendOrderCreatedMail_`、`sendPaymentReminderMail_`、`sendShippedMail_`），現在已經在 Part4 補齊。

### Q2: 需要重新部署嗎？
**A:** 是的，請將 Part4 的內容也貼到 GAS 編輯器中，然後重新部署。

### Q3: 如何測試寄信功能？
**A:** 在 GAS 編輯器中執行以下測試函數：
```javascript
function testEmail() {
  MailApp.sendEmail(
    'kensu.water.chestnut@gmail.com',
    '測試信件',
    '如果收到這封信，代表 MailApp 正常。'
  );
}
```

## 📧 郵件類型說明

1. **訂單成立信**（客人）
   - 主旨：`啃酥批發｜訂單成立＆匯款資訊｜訂單編號`
   - 包含：訂單明細、匯款資訊、LINE 聯絡按鈕

2. **新訂單通知**（老闆）
   - 主旨：`啃酥批發 新訂單｜訂單編號`
   - 包含：訂單明細、開啟試算表按鈕

3. **催款信**
   - 主旨：`啃酥批發｜款項提醒｜訂單編號`
   - 觸發：款項狀態改為「待匯款」或「未匯款」

4. **出貨通知**
   - 主旨：`啃酥批發｜您的訂單已出貨｜訂單編號`
   - 觸發：出貨狀態改為「已出貨」或「已寄出」
