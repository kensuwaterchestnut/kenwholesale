# 🔧 後端與前端修正清單

## 已修正的問題

### 1️⃣ 後端問題（GAS）

#### ❌ 問題 1：應付金額欄位變成日期格式
**原因**：寫入時沒有強制轉換為數字
**修正**：
```javascript
// 修正前
'應付金額': total,

// 修正後
'應付金額': Number(total) || 0,
```

#### ❌ 問題 2：明細工作表的「姓名」應該是「收件人」
**原因**：ITEM_HEADERS 使用「姓名」而非「收件人」
**修正**：
```javascript
// 修正前
const ITEM_HEADERS = [
  '建立時間','訂單編號','姓名','Email',...
];

// 修正後
const ITEM_HEADERS = [
  '建立時間','訂單編號','收件人','Email',...
];
```

並且在下單時也改為寫入收件人：
```javascript
// 修正前
'姓名': buyer,

// 修正後
'收件人': recipient,
```

#### ❌ 問題 3：出貨狀態改為已出貨後，明細沒有自動更新
**原因**：程式碼已經有 `updateDetailByOrder_` 函數，應該會自動更新
**檢查**：確認 `onEdit` 函數中的邏輯正確

### 2️⃣ 前端問題（index.html）

#### ❌ 問題：圖片無法顯示（即使檔名一模一樣）
**原因**：
1. 試算表的 `imgFilename` 可能包含副檔名（如 `啃酥專用粉.jpg`）
2. 雲端檔名也包含副檔名（如 `啃酥專用粉.jpg`）
3. 原本的配對邏輯不夠完善

**修正**：改進 `matchProductImage` 函數
```javascript
// 新的配對邏輯：
1. 移除試算表 imgFilename 的副檔名
2. 移除雲端檔名的副檔名
3. 進行完全匹配
4. 進行包含匹配（雙向）
5. 最後用 product.id 配對
```

**範例**：
- 試算表：`imgFilename = "啃酥專用粉.jpg"` 或 `"啃酥專用粉"`
- 雲端檔名：`啃酥專用粉.jpg`
- 結果：✅ 成功配對
